# docs-lint Exclusions

Owner: Engineering
Status: active
Last verified: 2026-06-04

What `docs-lint` deliberately does *not* flag, and why. Maintained for security review and enterprise audit — every entry here represents a check that does not run.

## Transient-directory exclusion list

`docs-lint`'s "should not be checked in" pattern check excludes the directories listed below. Each is a transient cache or build artifact that is already excluded by `.gitignore` per ecosystem convention; duplicating the exclusion in lint produced recurring blockers on every tool invocation that wrote a cache directory.

Source of truth: `LINT_EXCLUDED_TRANSIENT_DIRS` in `.wavefoundry/framework/scripts/wave_lint_lib/core_validators.py`. The exclusion list is a frozen Python constant — modifying it requires a framework change with a council-reviewed rationale.

| Pattern | Generated by | Why excluded |
|---|---|---|
| `__pycache__/` | Python interpreter (every `import` of a `.py` file) | Universal; gitignored by convention. The MCP server flow generates it on every Python invocation. |
| `.pytest_cache/` | `pytest` test runner | Universal in Python projects with tests. Per-run cache, not source. |
| `.mypy_cache/` | `mypy` type checker | Per-run cache. Re-generated on next mypy pass. |
| `.ruff_cache/` | `ruff` linter | Per-run cache. |
| `.tox/` | `tox` automation tool | Per-environment virtualenv state. Multi-MB; never appropriate to commit. |
| `.coverage` | `coverage.py` | Run-specific data file. (Listed for completeness — this is a file pattern, not a directory.) |

## What `docs-lint` still flags

The exclusion above is **targeted**, not a blanket bypass of "should not be checked in" classification:

- **Retired root wrappers** — `package-wave-framework`, `install-wave-framework`, `upgrade-wave-framework`, and other legacy launcher *executables* at the repository root still fail lint when found. See `FORBIDDEN_ROOT_WRAPPERS_RETIRED` in `wave_lint_lib/constants.py`. This denylist is a persistent tombstone (it guards against stale launcher files), not a prompt-surface artifact — do not confuse it with the documented legacy phrase alias **`Package wave framework`** → **`Package Wavefoundry`**, which is an intentional backward-compat routing entry in `docs/prompts/index.md`.
- **Relocated root wrappers** — `docs-lint`, `docs-gardener` etc. at the repository root still fail lint when found; their canonical location is `.wavefoundry/bin/`. See `FORBIDDEN_ROOT_WRAPPERS_RELOCATED`.
- **All other doc-shape checks** — wave-record headers, change-doc Decision Logs, persona doc structure, agent role docs missing `Role:` / `Category:` frontmatter, prompt-file extensions, journal salience markers, AC progression rules, wave dependency consistency, and dozens of other validators. Run `docs-lint --help` or read `wave_lint_lib/` to enumerate.

## Adding to the exclusion list

A new pattern joins `LINT_EXCLUDED_TRANSIENT_DIRS` only when:

1. The pattern is already covered by `.gitignore` in every repository where lint runs (so committing it would require explicit `git add -f` — a deliberate operator action, not a default-path mistake).
2. The pattern is generated on every routine tool invocation (`__pycache__` per Python import; `.mypy_cache` per type-check; etc.) such that lint failures would block routine workflows.
3. The change includes an entry in this doc with the generated-by tool, why-excluded rationale, and the date of admission.

These three conditions together prevent the exclusion list from becoming a permissive bypass. The list is intentionally short.
