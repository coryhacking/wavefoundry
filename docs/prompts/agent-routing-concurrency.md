# Agent Routing and Concurrency

Owner: Engineering
Status: active
Last verified: 2026-04-28

## Read-Only vs Write-Owning Lanes

**Read-only lanes** may run concurrently with other read-only lanes:
- Inventory and audit subagents
- Review lanes that only read (architecture-reviewer, security-reviewer reading docs)
- Discovery and planning passes

**Write-owning lanes** require exclusive access to their owned paths:
- Implementation lanes that edit repository code or docs
- Prompt surface renders
- Manifest updates

Only one write owner per protected surface at a time. The coordinator must verify each delegated write task matches the intended content before accepting the result.

## Protected Surfaces (single write owner)

| Surface | Rule |
|---------|------|
| `.wavefoundry/framework/seeds/` | Single write owner; requires `seed_edit_allowed` guard approval |
| `docs/prompts/` | Single write owner; requires `framework_edit_allowed` guard approval for broad changes |
| `AGENTS.md` | Single write owner for framework-maintenance edits |
| Hook configs (`.claude/settings.json`, `.cursor/hooks.json`, `.github/hooks/hooks.json`) | Generated by `render_platform_surfaces.py`; do not hand-edit in parallel |
| `docs/prompts/prompt-surface-manifest.json` | Single write owner; updated during init/upgrade/package |
| `docs/workflow-config.json` | Single write owner |

## Coordinator Responsibility

Before delegating a write task, the coordinator must:
1. Name the owned paths, forbidden paths, and expected deliverable for the subagent
2. After the subagent reports edits, re-read each changed file and verify the content matches the intended change

## Serialization Points

| Point | When to Serialize |
|-------|-----------------|
| `.wavefoundry/framework/VERSION` stamp | Only during packaging (`build_pack.py`); no concurrent writes |
| Manifest `framework_revision` | Must match `.wavefoundry/framework/VERSION` after init/upgrade; update atomically |
| Wave record status changes | Coordinator changes status (planned → active → completed); no concurrent status writes |
