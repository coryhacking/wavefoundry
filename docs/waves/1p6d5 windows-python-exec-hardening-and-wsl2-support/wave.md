## Objective

Two platform-support thrusts from the Windows-compat audit, deliberately scoped to exclude the launcher/Area-1 work. `1p6d6` (bug): harden the Python **execution** layer against native Windows NOW as forward-compat — fix the verified POSIX-isms (`build_pack` venv-path + `os.execv`, the unguarded `_pid_is_running`, the missing reindex `creationflags`, the `ntpath` backslash map-href, missing `encoding=`, read-only rmtree, symlink-only cache checks) as pure-additive `os.name` branches and cross-platform-safe fixes, with **zero POSIX/WSL2 behavior change** and `os.name`-mocked unit tests. `1p6d7` (doc): bless **WSL2 as a first-class supported Linux target** — WSL2 is Linux and already runs the working POSIX path, so this is an explicit platform-support statement + WSL2 gotchas + a smoke checklist, no code. When this wave closes, the execution layer is native-Windows-ready (verification deferred to a future Windows-smoke wave) and WSL2 is documented + validated as supported.
