## Objective

Make the scaffolded-but-empty `docs/design-system/` contract real and consumable. Today the token contract is empty (`tokens/*.json` are `{}`, `manifest.json extractedAt: null`) and the one real foundation doc is drifted from the shipped dashboard. This wave **extracts** the design tokens from the live `dashboard.css` evidence and reconciles the narrative (`1p6z6`), adds a **build pipeline** that emits framework-specific token exports (`12atj`), and **abstracts** the dashboard's reusable primitives into an exportable, token-bound module the dashboard consumes (`1p72v`). When it closes, agents have a populated, validated token contract + a primitive library — the "library first" base the claude.ai/design sync (follow-on wave) will build on.
