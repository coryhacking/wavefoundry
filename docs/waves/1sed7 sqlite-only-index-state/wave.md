## Journal Watchpoints

- Blocking design gate: complete the production/test `meta.json` consumer census and the SQLite build-generation/commit-order contract before the first implementation edit.
- Single-authority invariant: no steady-state JSON read/write/export or JSON-success/SQLite-failure mode may survive; legacy JSON is removed only after verified SQLite/Lance convergence.
- Cross-engine invariant: commit a `FULL`-equivalent durable SQLite building/dirty epoch before the first Lance mutation; fail readers closed until reconciliation and the attempt-ID-matched final metadata/generation/complete transaction. Lance remains authoritative for chunks; do not claim cross-engine atomicity.
- Reader invariant: every semantic/vector/FTS operation validates the same complete epoch token before and after all indexed work; a changed or incomplete token discards results, invalidates cached handles, and must not serve Lance/FTS as current.
- Writer census invariant: normal/scoped builds, zero-change reap/heal, standalone FTS rebuild, optimize/compact/rewrite, setup, upgrade, and MCP/close-triggered mutations all hold the build lock and epoch.
- Reset invariant: a whole-store reset escalates a docs-only/code-only request to all-layer convergence; do not manufacture untouched-layer model/chunker/content-version state.
- Scheduling constraint (RESOLVED by operator decision 2026-07-12): this wave implements FIRST; wave `1seav` was reconciled to consume the store API this wave defines (`1sbxq`'s freshness helper is now specified store-first with cache invalidation keyed on this wave's build generation, and `1sear`'s frozen-run signature is the store build generation) — the `server_impl.py` overlap is resolved by API direction, not implementation order deference.
- Compatibility watchpoint: upgrade from a pre-store/legacy-JSON installation must converge by reconstruction, not by retaining JSON as authority.
