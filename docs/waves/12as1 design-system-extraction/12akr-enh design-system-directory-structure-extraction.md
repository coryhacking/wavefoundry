## Rationale

Wavefoundry seeds already bootstrap `docs/design/design-language.md` and `docs/design/index.md` (see `seed-040` task 13), but there is no machine-readable extraction contract for tokens, components, assets, or gap reporting. Target repositories end up with narrative-only design docs and no deterministic surface for agents to consult when implementing UI. This change lands the **core extraction contract** under `docs/design/` so tokens and component specs become first-class, searchable artifacts — without silent invention when source evidence is missing.

This plan is the first of three splits from the original unified design-system extraction proposal:

- **This change (A)** — canonical tree, tokens, components, `manifest.json`, `gaps.md`, install/upgrade backfill, docs indexer support, rollback, architecture doc, and coexistence with the existing `design-language.md`.
- **Split B** — deep pattern and surface contract: `patterns/{navigation,feedback,data,trust}`, extended `foundations/`, deep accessibility (`keyboard.md`, `screen-reader.md`), `state-patterns/`, `validation-patterns/`, content depth, motion, borders/focus tokens. See `docs/plans/12arn-enh design-system-pattern-and-surface-depth.md`.
- **Split C** — bootstrap and governance: no-design-system visual-bootstrap path, multi-surface targets, HIG reference versions, deprecation/lineage fields, conditional product-class extensions (email/print/offline/notifications). See `docs/plans/12arn-enh design-system-bootstrap-and-governance.md`.

B and C depend on the tree and `manifest.json` schema landed here.

## Requirements

1. **Canonical output root.** All extracted artifacts land under **`docs/design/`**. The extraction tree must coexist with `docs/design/design-language.md` and `docs/design/index.md` (both already owned by `seed-040`) without overwriting them except as described in Requirement 10.
2. **Baseline tree (core subset).** Seeded contract must define the following required paths under `docs/design/` (deep subtrees owned by Split B are listed for awareness only and not required by this change):

   ```text
   docs/design/
   ├── README.md                       # human entry for the extraction tree
   ├── DESIGN.md                       # Google Labs DESIGN.md — agent-optimized distillation of the design contract; distinct from the operator-owned design-language.md body
   ├── AGENTS.md                       # agent contract for this subtree
   ├── manifest.json                   # machine-readable extraction index
   ├── VALIDATION.md                   # checks run / outcomes
   ├── gaps.md                         # required gap log
   ├── tokens/
   │   ├── primitives.tokens.json
   │   ├── semantic.tokens.json
   │   ├── components.tokens.json      # optional convenience layer
   │   ├── modes/
   │   │   ├── light.tokens.json
   │   │   └── dark.tokens.json
   │   └── README.md
   ├── exports/
   │   ├── README.md                   # generated by framework; explains each subdir and links to token-build pipeline
   │   ├── css/
   │   ├── tailwind/
   │   ├── ts/
   │   └── json/
   ├── components/
   │   ├── _index.json
   │   └── <component-name>/
   │       ├── spec.json
   │       ├── README.md
   │       ├── anatomy.md
   │       ├── tokens.json
   │       └── visuals/
   ├── foundations/
   │   ├── color.md
   │   ├── typography.md
   │   ├── spacing.md
   │   ├── radius.md
   │   ├── elevation.md
   │   └── motion.md
   ├── accessibility/
   │   ├── contrast-report.json
   │   └── README.md
   └── docs/design/
       ├── version.json
       ├── source-map.json
       └── proposed-additions.md       # agent escape valve: new component proposals before they're built
   ```

   Split B adds `patterns/`, `state-patterns/`, `validation-patterns/`, `content/`, `skills/`, extended `foundations/`, deeper `accessibility/`, `icons/`, `illustrations/`, `logos/`, `images/`, and additional token files (`borders.tokens.json`, `focus.tokens.json`, `z-index.tokens.json`, `motion.tokens.json`). Split C adds multi-surface and governance fields to `manifest.json` / `components/_index.json`.
3. **DTCG two-tier token contract.** Tokens must follow DTCG-style primitive + semantic split. `primitives.tokens.json` holds raw values; `semantic.tokens.json` resolves to primitive references. `components.tokens.json` is an **optional** third layer for component-scoped aliases — it is not a third tier of design decisions, only a convenience index. Mode files under `tokens/modes/` must cover light and dark as first-class modes (not cosmetic inversion — dark mode must be verified for contrast and hierarchy, flagged in `gaps.md` when inadequate).

   **Token naming convention (mandatory).** All token names must follow the `{category}.{subcategory}.{scale}.{variant}` dot-path schema (e.g. `color.primary.500`, `color.action.primary.background`, `spacing.4`, `radius.button`). Leaf names must match the evidence source exactly at the token level; if a source name must be normalized (Figma path vs DTCG path), both the original and normalized names must be recorded side-by-side in `source-map.json`. This convention is the shared contract Split B and Split C extend without renaming. Validators added in this change must reject tokens whose names do not match the dot-path schema.
4. **Component `spec.json` core fields.** The core schema has two layers:

   **Identity fields (required, owned by this change):** `id` (kebab-case identifier, e.g. `"button"`), `name` (display name), `category` (e.g. `"actions"`, `"inputs"`, `"feedback"`, `"navigation"`, `"layout"`, `"data-display"`), `status` (`"stable"` | `"beta"` | `"deprecated"`), `description` (one sentence, plain language), `figma` (object with `nodeId`, `componentSetId`, `url` — `null` when no Figma source), `codeConnect` (`null` when no Code Connect mapping), `anatomy` (array of named regions; may be empty), `variants` (array of variant names), `props` (array of prop definitions), `slots` (array of slot definitions), `tokens` (array of token IDs used by the component), `doNotUse` (array of plain-English agent guardrails, e.g. `"Don't use more than one primary button per visual section"`; empty array when none), `preferOver` (array of component IDs this component supersedes for common cases; empty array when none).

   **Behavioral fields (reserved, populated by Split B):** `states`, `responsive`, `motion`, `accessibility`, `content` — the core seed emits these keys with `null` values so Split B is a purely additive population of existing slots. Validators in this change verify the keys are present (even when `null`); Split B validators verify values are populated where source evidence exists.
5. **`manifest.json` core schema.** Required fields: `schemaVersion` (**semver string**, e.g. `"1.0.0"`; upgrade reconciliation in `seed-160` compares this field using semver ordering — do not use date-based or integer formats), `extractionVersion`, `extractedAt` (ISO-8601), `canonicalRoot` (must equal `docs/design`), `sourceStrategy` (enum: `figma-extract`, `repo-evidence-only`, `visual-bootstrap`, `hybrid`; full semantics for `visual-bootstrap` owned by Split C), `evidenceTypes` (array), `artifactCounts` (tokens/components/etc.), `modes` (array, must include `light` and `dark`), `validationSummary` (counts of passing/failing checks). Additional fields (`targetSurfaces`, `platformStandards`, `deprecations`) are reserved for Split C — the schema must accept but not require them.
6. **`gaps.md` required with categorized, severity-ranked entries.** Categories (core subset): `tokens`, `components`, `assets`, `content`, `accessibility`, `responsive`, `motion`, `states`, `meta`. Severities: `critical`, `important`, `nice-to-have`. Each entry must include: category, severity, source searched, missing item, recommended action. A summary count by severity and category is required at the top of the file. Split B extends the category list (`patterns`, `navigation`, `shell`, `feedback`, `data`, `trust`, `i18n`, `locale`, `governance`).
7. **Extract, don't invent.** Missing-source values must be represented as explicit `null` in tokens/specs plus a matching `gaps.md` entry. Silent defaults are forbidden. Each extracted item must record an evidence confidence (`high`, `medium`, `low`) in `source-map.json`; low-confidence items must produce a `gaps.md` entry.

   **`source-map.json` core schema.** Each entry in `source-map.json` must record the following fields: `id` (the dot-path token name or `<category>/<item-id>` for non-token artifacts), `sourceType` (one of `figma`, `code`, `visual`, `inferred`), `confidence` (`high`, `medium`, or `low`), `sourceRef` (URI, file path, or Figma node ID; `null` if no traceable source), `normalizedFrom` (the original Figma/source name when it was renamed to meet the dot-path convention; omit if unchanged), and `platformHint` (optional surface tag used by Split C for multi-surface delta recording). Split B and Split C extend this schema by populating `platformHint` and adding new `sourceType` values — neither adds nor renames the core fields.
8. **Install behavior (`seed-010`).** During **Init wave framework**, if `docs/design/` is missing **or** any core required path from Requirement 2 is absent, the install step must create the missing paths with empty placeholders (`manifest.json` with `sourceStrategy: "repo-evidence-only"` and empty `artifactCounts`; `gaps.md` with a header and empty category sections; index files as empty JSON objects where applicable). Install must be **merge-safe**: never delete or overwrite existing files, including `design-language.md` or any operator-customized artifact.
9. **Upgrade behavior (`seed-160`).** During **Upgrade wave framework**, the same backfill logic as install runs — create missing paths, never overwrite. In addition, upgrade must compare existing `manifest.json` against the current `schemaVersion`; when schema fields are missing, upgrade must **add** them with null/empty values and record a `meta`-category entry in `gaps.md` rather than rewriting operator data.
10. **`design-language.md` coexistence rule and `DESIGN.md` role.** Extraction **never rewrites** `design-language.md` or `docs/design/index.md` body content. Extraction may only: (a) append a cross-link row to `index.md` listing new extraction artifacts with status `generated`; (b) add a "See extracted contract" pointer at the top of `design-language.md` when it does not already exist. Both operations must be idempotent. This rule applies to both install and upgrade paths.

   **Role distinction:** `design-language.md` is the **operator-owned narrative** design document (seeded by `seed-040 task 13`; reflects intent, rationale, and departures). `DESIGN.md` is the **agent-optimized distillation** in Google Labs DESIGN.md format — generated from the extraction contract, under 400 lines, YAML front-matter holds token references, markdown body holds rationale distilled for agents. `DESIGN.md` is regeneratable; `design-language.md` is not. Seeds and validators must never conflate the two.
11. **Install/upgrade backfill checklist (testable).** Seed prompts must include an explicit checklist format: each required path from Requirement 2 appears as a checkbox line (`- [ ] docs/design/manifest.json`). Completing the step means all boxes are checked. Framework tests under `.wavefoundry/framework/scripts/tests/` must include at least one test that (a) runs the backfill logic against a temp target without `docs/design/`, (b) asserts every required path now exists, and (c) asserts `design-language.md` body is byte-identical if it already existed.
12. **Docs indexer — treat design JSON as doc chunks.** `.wavefoundry/framework/scripts/chunker.py` must route paths matching `docs/design/**/*.json` through the markdown/doc chunker path (or a new JSON-as-doc branch) so these files produce `doc` kind chunks and are findable via `docs_search` on a docs-only index build. The chunker must still fall back to line-window chunking for malformed JSON. Tests must cover: a token file produces doc chunks; a malformed JSON file does not crash; files outside `docs/design/` keep their existing routing.
13. **Rollback / clean re-extract.** Seeded guidance must document a clean re-extraction path: (a) move the existing `docs/design/<subtree>` to a timestamped backup directory (`docs/design/.backup/<ISO-date>/`) under `docs/design/`; (b) regenerate; (c) diff against the backup for operator review. The process must never auto-delete operator artifacts without the backup step. A `meta`-category `gaps.md` entry must be written when a backup is created.
14. **Architecture doc impact.** A new `docs/architecture/design-system.md` hub doc must be seeded describing: extraction philosophy (machine-readable vs narrative), where `design-language.md` fits, when extraction regenerates vs operator edits, and the relationship between `docs/design/` and the semantic index. Cross-linked from `docs/ARCHITECTURE.md` and `docs/design/index.md`.
15. **Prompt-surface cross-references.** Update `seed-010`, `seed-040`, `seed-050`, `seed-100`, `seed-150`, `seed-160`, and `docs/README.md` (when seeded) so `docs/design/` extraction is discoverable and the install/upgrade contract is consistent across seeds. No new operator shortcut phrases are introduced by this change.
16. **Storybook out of scope.** Base extraction contract must not require Storybook-specific files (`stories.meta.json`, `storybook/*`, MDX catalogs). Storybook remains opt-in follow-up.
17. **`docs/design/AGENTS.md` content contract.** The seeded `AGENTS.md` must specify the following agent rules (not just exist as a stub): (a) check `components/_index.json` before building any UI component — use existing, don't create a duplicate; (b) if no match exists, append a proposal to `proposed-additions.md` rather than inventing inline; (c) check `tokens/` before writing any hard-coded value — reference semantic tokens only, never primitives or raw hex/px/z-index/duration; (d) format: under 200 lines, plain rules, no prose summaries. Split B will extend this file with microcopy and icon lookup rules when those subtrees land. The seed must state which sections Split B will add so the file is clearly provisional.
18. **Core token validators.** The validator module added in this change must also include: (a) **broken reference check** — every `{token.path}` alias in `semantic.tokens.json` resolves to a key in `primitives.tokens.json`; unresolved refs are lint failures; (b) **orphan primitive check** — every primitive token is referenced by at least one semantic token, or carries `"$extensions": { "primitive-only": true }`; orphans are `important`-severity warnings; (c) **mode parity check** — `tokens/modes/light.tokens.json` and `tokens/modes/dark.tokens.json` have an identical key set; missing keys in either direction are lint failures. These validators belong here (not Split B) because the core seed seeds the initial token and mode files — a broken or mode-mismatched extraction must fail before Split B ever runs.

## Scope

**Problem statement:** Target repositories seeded by Wavefoundry have narrative design docs but no deterministic, machine-readable design surface. Agents implementing UI must infer tokens and component shapes from prose or code, and there is no enforced gap log when source evidence is missing. Without the core tree and schema, Splits B and C cannot layer on patterns, deep a11y, multi-surface recording, or governance metadata.

**In scope:**

- Define the core `docs/design/` tree (Requirement 2) and commit it to seed prompts.
- Define `manifest.json` core schema with `schemaVersion`, `sourceStrategy`, `canonicalRoot`, `extractedAt`, `artifactCounts`, `modes`, `validationSummary` — plus reserved fields for Splits B and C.
- Define `gaps.md` core categories and severities.
- Define `spec.json` core + reserved fields for Split B.
- Install/upgrade backfill with merge-safe behavior and a testable checklist.
- Rollback / clean re-extract path via timestamped backup directory.
- `chunker.py` change so `docs/design/**/*.json` is indexed as doc chunks, plus tests.
- `docs/architecture/design-system.md` hub doc.
- Seed cross-reference updates and `docs/README.md` row.
- Evidence confidence model (`high`/`medium`/`low`) in `source-map.json`.
- `docs/design/AGENTS.md` seed content (four agent rules; provisional Split B extension stubs).
- `spec.json` full identity field set: `id`, `category`, `status`, `description`, `figma`, `codeConnect`, `doNotUse`, `preferOver` — plus reserved behavioral fields as `null`.
- Core token validators: broken reference, orphan primitive, mode parity, dot-path naming, `_index.json` ↔ folder parity.
- `proposed-additions.md` stub.
- `exports/` stub directories with explicit out-of-scope note in seed and README.

**Out of scope (owned by Split B):**

- `patterns/{navigation,feedback,data,trust}/` subtrees and `foundations/shell.md` / `density.md` / `data-visualization.md` / `media-motion.md`.
- Deep accessibility (`keyboard.md`, `screen-reader.md`), `state-patterns/`, `validation-patterns/`.
- Content depth (`voice.md`, `microcopy.json`, `formatting.md`, `i18n.md`, `rtl-layout.md`, `locale-formats.md`, `brand-legal.md`).
- Extended token files (`borders.tokens.json`, `focus.tokens.json`, `z-index.tokens.json`, `motion.tokens.json`) and dedicated motion foundations depth.
- Asset contract (`icons/`, `illustrations/`, `logos/`, `images/`) and the semantic validation rules that operate on them.
- `skills/` tree.

**Out of scope (owned by Split C):**

- Multi-surface recording (`targetSurfaces`, platform HIG cross-checks).
- No-design-system visual-bootstrap path (reference URLs, captures, decks).
- `platformStandards[].referenceVersion` for HIG drift tracking.
- Deprecation/lineage (`deprecated`, `supersedes`, `sunset`) on `manifest.json` and `components/_index.json`.
- Conditional product-class extensions (email, print, offline, notifications) and their worked examples.
- Best-practice proposal appendices in `gaps.md` with starter palettes/type scales.

**Also out of scope:**

- Storybook-specific metadata and MDX catalogs.
- Token-build pipeline outputs — `exports/css/`, `exports/tailwind/`, `exports/ts/`, and `exports/json/` subdirectory *contents* (CSS custom properties, Tailwind theme config, TypeScript token types, flat JSON) are a follow-on wave (`12atj-feat design-token-build-pipeline`) requiring a build tool (Style Dictionary or equivalent). The framework generates the `exports/` folder, its four subdirectories, and `exports/README.md` (which explains each format and links to the pipeline plan) — but the generated token files inside those subdirectories are out of scope for this wave.
- Framework-specific code generation.
- Installing UI tooling or build pipelines.
- Redesigning upstream Figma sources instead of extraction with gaps.
