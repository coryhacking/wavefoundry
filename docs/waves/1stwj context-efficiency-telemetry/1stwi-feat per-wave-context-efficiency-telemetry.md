## Rationale

Our retrieval tools are a core token-efficiency value: they return bounded cited chunks instead of the agent reading whole files. Typed lifecycle tools provide a second efficiency signal by compressing long public workflow instructions into deterministic checks and bounded responses, although a tool call cannot prove that the mapped prompt was never independently loaded or that every prompt obligation was automated. We do not currently quantify either class. A survey of `agentmemory`'s token-savings surface showed its "~92% saved" is a marketing gauge — `obsCount × 80` (a hardcoded "full context" per-observation constant) vs a `min(obsCount,50) × 38` injected cap, i.e. `count × magic-number` against an unbounded baseline that inflates the percentage the longer you use it, with no tokenizer and no real measurement (`src/cli.ts` `status`).

This change ships one conservative, auditable **estimated token savings** total per wave and stage. Retrieval credits each returned content or explicitly enumerated structural source version at most once per phase; every eligible tool request and response remains a debit, and a source counts again only after a phase transition or version change. Typed lifecycle tools contribute one mapped workflow-prompt gross credit plus their actual interaction debits. Saved model output, deterministic artifact production, reduced reasoning, and avoided investigation loops contribute only through a same-key versioned paired evaluation whose assisted run meets or exceeds baseline quality; absent qualifying evidence, that residual is zero. Tool-call reduction remains supporting evidence rather than an independently additive multiplier. Every eligible call either writes its bounded event and deduplication decision to the host-local SQLite authority or durably poisons positive projection before returning; if neither write is possible, it returns `telemetry_persistence_failed`. Lifecycle, reload, and upgrade boundaries project already-durable totals into `wave.md`. Telemetry does not influence retrieval, ranking, lifecycle outcomes, or gating except for that explicit undurable double-failure result.

## Requirements

1. **Retrieval per-call evidence and baseline classes.** Each eligible call estimates tokens with `ceil(UTF-8 byte length / 4)`. The request debit is canonical compact JSON of the public arguments and the response debit is canonical compact JSON of the complete core response before telemetry. Each distinct content-bearing or explicitly enumerated structural source contributes the strongest contained gross baseline: same-version verified current file size; readable current-file size estimate; or an already-captured/indexed expected size estimate when the current file cannot be read. Sources with no size-bearing baseline contribute zero. Telemetry never hashes or reads whole-file contents solely to improve classification. The per-call committed decision reports gross source credit and request/response debits; the stage headline is calculated only by Requirement 13's closed ledger.
2. **Retrieval tool coverage.** Emit `context_avoided` `{estimated_request_tokens, estimated_returned_tokens, estimated_source_tokens, estimated_avoided_tokens, source_files_counted, source_files_verified, source_files_estimated, captured, persistence, method}` on: `code_ask`, `code_search`, `code_lexical`, `docs_search`, `code_keyword`, `code_pattern`, `code_constants`, `code_read`, `code_outline`, `code_definition`, `code_references`, `code_callhierarchy`, `code_impact`, `code_dependencies`, `code_callgraph`, `code_graph_path`, and `code_graph_community`. `estimated_avoided_tokens` is the call's committed net contribution after atomic source-version deduplication and both interaction debits, clamped at zero for display; durable stage reconciliation retains negative call contributions. `source_files_counted = source_files_verified + source_files_estimated`. `persistence` is `durable` or `failed`; a failed write activates Requirement 7's accounting-gap barrier. Path-list-only tools such as `code_list_files` are excluded.
3. **Lifecycle workflow-input component.** `wave_create_wave`, `wave_prepare`, `wave_implement`, `wave_review`, and `wave_close` emit an auditable workflow-input component derived from exactly one contained project-local prompt (`create-wave`, `prepare-wave`, `implement-wave`, `review-wave`, or `close-wave`); missing/unreadable prompt contributes zero with no fallback. Every invocation that reaches the registered public handler receives a unique invocation ID and contributes its request/response debits once, including lifecycle no-ops and handler-level validation/refusal responses. Only a genuine completed milestone contributes the mapped prompt gross credit; implementation/tests, transitive docs, validator scans, reviewer reasoning, nested helpers, pre-handler MCP schema rejection, no-ops, and failed milestones contribute zero prompt credit. Retrying persistence for the same event retains its ID and cannot duplicate it; a later genuine cycle receives a new ID. The prompt baseline is an upper-bound instruction surface—not proof that it would otherwise have entered model context—so its provenance remains machine-readable even though the human projection combines it into the one conservative total.
4. **Returned-source eligibility.** A file contributes only when the response carries content or structural facts derived from it. Content-bearing examples include chunks, excerpts, definitions, constants, matched lines, and call-site snippets. For graph/navigation results, only distinct files needed to derive the returned symbols, edges, impact set, path, or community contribute; corpus membership, hidden fan-out, and files not represented in the public result remain inert. The structural baseline is explicitly an estimated counterfactual file-read scope and uses the same phase/source-version deduplication as content retrieval.
5. **Process-scoped attribution with durable phase identity.** Each `ImplHandler` owns a random `producer_id` and focus `{wave_id, stage, phase_id, paused_stage}` guarded for in-process concurrency. No repository-global focus exists. Each genuine lifecycle transition mints a monotonic wave-owned `phase_id` (for example `review-2`); a later return to the same named stage is a new phase. Every telemetry event captures that exact phase ID, or a durable producer-scoped `_general` identity before wave association. Within one process, attribution is explicitly "last lifecycle focus association," not causal exclusivity. The random producer ID contains no host/user/PID data and exists only to let create/prepare transfer that producer's durable general events without capturing another MCP process.
6. **General bucket + create/prepare association.** Before a wave association exists, every eligible call writes to a durable producer-scoped `_general` phase. Successful `wave_create_wave(create)` and mutating `wave_prepare(ready|create)` atomically reassign only the invoking producer's unassigned general events to that wave's `pre-wave` phase. Dry-run/failed lifecycle operations do not transfer; empty/replay is a no-op. The fixed note remains *"associated unattributed context at successful wave creation or preparation; may include exploration not exclusive to this wave."* Other producers' general work remains isolated and cannot be captured.
7. **SQLite write-through authority + accounting-gap barrier + wave projection.** `.wavefoundry/logs/context-efficiency.sqlite` is the live host-local authority for events, atomic source/version deduplication, lifecycle invocation replay, paired-evaluation attachments, phase state, and producer-scoped general attribution. Every eligible public call attempts one bounded transaction before returning; after commit, a process crash cannot lose that event. If the transaction still fails after bounded busy retry, the runtime durably poisons publication through an opaque zero-content `.wavefoundry/logs/context-efficiency.gap` sentinel and, when SQLite becomes writable, imports the poison into store state. While either poison is present, `wave_current`, `wave_audit`, and projection report `accounting_gap`, suppress the positive headline, and never silently publish prior credits without the missing debit. No automatic clear exists because the missing event cannot be reconstructed. If both the event transaction and durable sentinel write fail, the registered tool does not return the otherwise-successful core result; it returns a stage-named `telemetry_persistence_failed` envelope, so no unrecorded successful eligible interaction exists. The store and sentinel persist no paths, queries, returned content, prompts, secrets, host, PID, or user. Successful mutating `wave_create_wave(create)`, `wave_prepare(ready|create)`, `wave_implement(create)`, `wave_pause(create)`, bare `wave_reopen`, `wave_close(create)`, `wave_mcp_reload`, and `wave_upgrade` project already-durable totals into the marker-owned `wave.md` checkpoint under the existing project-global writer lock. Projection failure never rolls back committed telemetry and remains retryable.
8. **Conflict-safe durable checkpoint.** `wave.md` carries a marker-delimited `## Context Efficiency` snapshot with one plain per-stage table using only `Stage`, `Calls`, and `Estimated token savings`, plus a total row. Methodology stays in surrounding prose. Per-call envelopes retain raw candidate evidence; the durable state comment retains the store instance ID, content-source credit, structural-source credit, workflow-prompt credit, request debit, response debit, matched-pair residual, quality-gate, and source-credit count/drop state. All framework `wave.md` writers in the touched lifecycle census share one project-global cross-process serialization helper; telemetry projection re-reads current bytes under that lock, replaces only its marker block atomically, and never overwrites unrelated status/evidence/operator edits. Projection failure leaves lifecycle success unchanged and durable telemetry pending for retry. If a checkpoint's SQLite authority is absent, reset, or has a different store instance ID, that wave enters `credit_history_unavailable`: the checkpoint remains visible, new calls retain their interaction debits for diagnostics but earn no combinable credit/headline update, and no source identities or totals are reconstructed. Only restoration of the matching intact SQLite authority resumes combinable measurement.
9. **Surfacing + guardrails.** `wave_current`/`wave_audit` show the one durable estimated-savings total, auditable component provenance, SQLite health/generation, focus association, producer-scoped `_general`, pending projection state, counted source credits, and paired-evaluation quality/calibration status. They distinguish live SQLite authority from the last published `wave.md` snapshot. No bare "% saved," tokenizer-exact/model-billing claim, hidden graph fan-out, unlabeled current-file proxy, raw code-size credit, causal reasoning claim, or call-count multiplier is permitted.
10. **Telemetry isolation and annotation honesty.** Instrument exactly once at eligible public-envelope boundaries. Because write-through telemetry changes host-local operational state, telemetry-bearing retrieval/navigation tools must not claim MCP `readOnlyHint=true`; they use a non-destructive observational annotation that discloses the local SQLite write, while tools without telemetry remain unchanged. The registered `wave_review` path likewise discloses its telemetry event but still requests no background index refresh and validates adoption with `persist_adoption=False`. Telemetry calculation, a committed transaction, projection, prompt/stat lookup, or locking never changes the core status, diagnostics, results, ordering, `search_mode`, `fallback_reason`, lifecycle result, gate behavior, or approval annotations. The sole fail-closed exception is Requirement 7's undurable double failure: when neither the event nor accounting-gap poison can be persisted, the tool returns `telemetry_persistence_failed` instead of presenting an unaccounted successful result.
11. **Distribution, marker namespace, and compatibility.** Every framework-owned generated region emits the concise `wave:` namespace (`<!-- wave:<region> begin/end -->`, or `# wave:<region> begin/end` where the host format requires line comments). New output never emits `wavefoundry:` or `waveframework:` marker namespaces. Readers, chunk exclusion, marker metadata, and renderers recognize the two legacy namespaces for compatibility; a renderer touching a legacy owned region replaces it in place with the canonical `wave:` pair instead of appending a duplicate. `context_efficiency.py` owns one canonical marker-delimited carrier block shared by runtime wave scaffolding and `render_agent_surfaces.py`. The public `wf render-surfaces` path creates/updates exactly that owned region in the target Create-wave prompt while preserving all project prose outside it; seeds 100/110 carry the canonical bootstrap obligations rather than becoming an unrendered duplicate. Fresh install/package/upgrade deliver the implementation, all five project-local prompts, managed `.wavefoundry/logs/` ignore, and canonical markers without eager telemetry state. Upgrade does not bulk-rewrite historical `wave.md`/`events.jsonl`; an existing open wave or rendered owned surface migrates when its normal writer next touches that region.
12. **Independent-reference verification and budgets.** Fixtures use independent byte/stat/prompt/diff/SQL expectations, two real child processes, stale-index growth/shrink controls, and controlled concurrent wave projection. Structural budgets: an eligible call performs one canonical request encode, one canonical response encode, at most one metadata batch read, at most one stat per distinct returned source, zero whole-file reads/hashes solely for telemetry, and one bounded SQLite transaction; a projection boundary adds at most one serialized `wave.md` replacement. Repeated p95 budgets use Requirement 18's frozen-environment thresholds rather than one-shot timings.
13. **One closed ledger algebra.** Persist gross credits and interaction debits; never persist pre-netted components. For each stage: `direct_net = retrieval_source_credit + structural_source_credit + workflow_prompt_credit - request_argument_debit - response_debit`; `estimated_tokens_saved = max(0, direct_net + matched_pair_residual)`. Each eligible public call contributes its canonical compact-JSON argument tokens once to `request_argument_debit` and its complete pre-telemetry response tokens once to `response_debit`. Retrieval/structural source and workflow-prompt values are gross credits. `matched_pair_residual` is already net and excludes `direct_net`; it is never debited again. A numeric repeated-call fixture proves order-independent reconciliation and that no request or response is subtracted twice.
14. **Phase/source-version credit and state authority.** A source baseline is credited once per `(wave, phase_id, source_id, version_id)` across processes, regardless of whether it first appeared as content or structural evidence; provenance records `content`, `structural`, or `both` but never creates a second whole-file credit. `source_id = SHA-256(normalized repo-relative path)`; `version_id = SHA-256(canonical JSON of the available stat signature: size, normalized mtime, inode, plus any stronger already-computed version proof)`. Duplicate chunks, repeated calls, transaction retries, concurrent producers, and cross-class rediscovery do not re-credit it. A new phase ID or proven version change creates a new boundary. SQLite owns the atomic uniqueness row and retains completed-phase identities, so later calls and process restarts merge exactly without `wave.md` identity bloat. A phase persists at most 100,000 identities; excess source credit is dropped while interaction debits continue. The public per-call envelope reports the committed credit/debit decision; SQLite is the authority and `wave.md` is its portable numeric checkpoint.
15. **Structural graph source census.** Structural credit is limited to contained files explicitly represented in public results: `code_dependencies.data.path`; heuristic `code_impact.data.path` and `data.importers[*].file`; graph `code_impact.data.affected[*].source_file`; `code_callgraph.data.nodes[*].source_file`; `code_graph_path.data.path_nodes[*].source_file`; and `code_graph_community.data.nodes[*].source_file`. Missing/external paths, diagnostics, suggestions, hidden scan candidates, graph neighbors appended to another tool, edges without a returned file-bearing node, corpus membership, and truncated-away members contribute zero. Per-tool public-shape fixtures prove the census, cross-class once-credit rule, and known-bad hidden-fan-out exclusions.
16. **Matched paired evaluation and quality gate.** Saved model output, fewer tool calls, shorter reasoning, and deterministic artifact production enter the headline only through a pair set attached to that exact `{wave_id, phase_id, stage, task_spec_digest, repository_snapshot_digest, model_id, model_version, tool_configuration_digest}`. The shipped artifact is `.wavefoundry/framework/evals/context-efficiency-pairs.schema.json`; the scorer is `.wavefoundry/framework/scripts/score_context_efficiency_pairs.py`. Each arm records provider-reported input/output tokens, tool calls, completion, and blind pre-unmask rubric scores (correctness, completeness, evidence, maintainability; integers 0–4). A pair qualifies only when both arms complete and the assisted score is componentwise greater than or equal to baseline. At least five qualifying same-key pairs are required. For pair `i`, `total_delta_i = baseline_input_i + baseline_output_i - assisted_input_i - assisted_output_i`; `residual_i = max(0, total_delta_i - assisted_direct_net_i)`. The active phase attachment's `matched_pair_residual` is the minimum residual across qualifying pairs; the stage residual is the sum of active residuals for distinct phases in that stage. `evaluation_id` is globally single-target and cannot be attached to another wave or phase. Any key mismatch, score regression, missing provider usage, fewer than five qualifying pairs, or negative residual contributes zero. Input/output deltas and tool-call deltas remain auditable evidence but are not separately additive.
17. **First shipped schema and installation boundary.** This is the first shipped context-efficiency schema. No pre-release aggregate/checkpoint compatibility layer, legacy table, or historical telemetry payload is retained. Fresh install, package, and upgrade ship the current implementation, schema, scorer, attachment tool, and docs without eagerly creating telemetry state. An existing current-schema checkpoint whose matching store authority is unavailable follows Requirement 8's frozen `credit_history_unavailable` rule.
18. **Scale and write budgets.** Each eligible call performs one bounded SQLite transaction with batched `INSERT OR IGNORE` source credits plus one event/aggregate update; no `wave.md` write occurs on retrieval. A phase accepts at most 100,000 durable source identities, after which new source credits are dropped while interaction debits continue. Repeated p95 budgets are ≤10 ms for 10 candidates, ≤25 ms for 50, ≤75 ms for 1,000, and ≤250 ms for 10,000 on the frozen test environment. A competing-writer fixture holds the write lock across the initial attempt, proves bounded retry either commits exactly once or activates the accounting-gap barrier, and pins the failure-path latency ceiling at 1 second. Row-cap/drop/gap diagnostics are explicit and never alter the core tool result.
19. **Projection barriers, not timer flushes.** Because events are durable before each call returns, no inactivity timer, shutdown flush, or in-memory late-event recovery exists. `wave_upgrade` and `wave_mcp_reload` first project all pending durable generations for affected waves into `wave.md`; projection failure refuses the upgrade/reload so framework replacement cannot strand an unpublished schema generation. Other lifecycle mutations project their target wave after committing their own event. Process crash after transaction commit loses no event; a crash before commit returns no successful telemetry claim.
20. **Paired-result ingestion authority.** `wave_context_efficiency_attach_evaluation(wave_id, phase_id, report_path?, mode, applicability?)` is the sole public writer for paired-evaluation scope and scorer output. Before either run begins, `mode=register` requires the full applicability key `{wave_id, phase_id, stage, task_spec_digest, repository_snapshot_digest, model_id, model_version, tool_configuration_digest}`, verifies that `stage` is the authoritative stage recorded for `phase_id`, persists it as the immutable phase scope, and returns its digest. `mode=attach|replace` accepts only a contained schema-valid scorer report, recomputes the scorer result, and requires its exact key to equal that pre-registered durable scope rather than trusting report self-consistency. `evaluation_id` is replay-idempotent; conflicting reuse fails closed. A phase has at most one active attachment; replacement requires `supersedes_evaluation_id`, preserves the prior record as inactive, and recalculates rather than adds the residual. `mode=revoke` sets the active residual to zero while preserving chronology. Concurrent register/attach/replay/replace/revoke fixtures prove one immutable applicability authority, one active residual, and no double credit.

## Scope

**Problem statement:** We deliver measurable token savings through bounded retrieval and typed lifecycle automation and want one conservative per-wave/stage total that includes interaction costs, phase-aware source reuse, and only quality-gated paired-evaluation residuals.

**In scope (edited under `framework_edit_allowed`):**
- `.wavefoundry/framework/scripts/context_efficiency.py` — estimator, verified/current-file/captured-size baseline classification, phase-scoped source-version credit, gross-credit/debit ledger, project-local lifecycle-prompt contract, process-local focus, write-through SQLite authority, snapshot projection, and evaluation attachment.
- `.wavefoundry/framework/scripts/server_impl.py` — exactly-once envelope instrumentation for 17 retrieval tools and five lifecycle milestones; `wave_review` purity repair; shared lifecycle writer serialization; focus/write-through/projection wiring; reload/upgrade barriers; `wave_current`/`wave_audit` fields.
- `.wavefoundry/framework/scripts/render_agent_surfaces.py` — public-render propagation of the dedicated Context Efficiency carrier region sourced from `context_efficiency.py`, preserving project-authored prompt prose.
- `.wavefoundry/framework/scripts/chunker.py`, `gen_codebase_map.py`, `review_evidence.py`, and `server_impl.py` — canonical `wave:` emission plus bounded legacy marker recognition for chunk exclusion, generated maps/review projections, and `code_read.marker_regions`.
- `.wavefoundry/framework/scripts/wave_lint_lib` — marker uniqueness and projection-shape validation only.
- `.wavefoundry/framework/seeds/100-project-prompt-surface-bootstrap.prompt.md` and `110-wave-memory-bootstrap.prompt.md` — canonical new-wave/non-MCP projection anchors, edited under `seed_edit_allowed` and regenerated through the public render path.
- `.wavefoundry/framework/evals/context-efficiency-pairs.schema.json` and `.wavefoundry/framework/scripts/score_context_efficiency_pairs.py` — provider token-usage import, blind quality rubric, exact-key eligibility, five-pair minimum, minimum residual, and explicit zero when evidence is absent or ineligible.
- Docs — a reference contract for the single total and its auditable components, `docs/specs/mcp-tool-surface.md`, `docs/architecture/domain-map.md`, `docs/architecture/data-and-control-flow.md`, and `docs/architecture/testing-architecture.md`.
- Tests — verified/current-file/captured-size classification, phase/version deduplication, exact graph-source census, closed ledger algebra, paired-evaluation eligibility/attachment, exact retrieval/lifecycle censuses, prompt-map authority, invocation replay protection and repeated-cycle accumulation, real two-process isolation, per-call write-through/projection concurrency, store-identity fail-closed recovery, installation/packaging/upgrade compatibility, scale budgets, and independent-reference invariants.

**Out of scope:**
- **Memory re-exploration savings** — the biggest real saving (a `wave_memory_brief` that prevents a 20–100k-token rediscovery) is not per-call measurable; it stays OUT of this number and is at most noted qualitatively. (Handled conceptually by the separate memory capture-assist effort.)
- **Any dollar/cost/pricing figure** — tokens only; no model-price assumptions (explicitly avoiding `agentmemory`'s undocumented `$` claims).
- **Any change to retrieval behavior, ranking, chunking, or gating** — telemetry only.
- **A dashboard panel** for the number — optional later follow-on.
- **Cross-wave/global analytics or trends** beyond the separately labeled live `_general` bucket, create/prepare-associated `pre-wave` rows, and per-wave totals.
- **Hidden graph scope as savings** — only files explicitly represented in the public structural result may contribute; graph membership, internal scan fan-out, appended neighbors, and truncated-away nodes remain excluded.
- **Uncalibrated reasoning or loop savings.** Runtime telemetry cannot prove an avoided model turn; only a qualifying paired evaluation can contribute the conservative residual.
- **Unbounded "code replaced an agent" estimates.** Implementation/test size, validator scan fan-out, reviewer reasoning, nested helpers, and lifecycle/status utilities without qualifying same-key paired evidence are not counted.
