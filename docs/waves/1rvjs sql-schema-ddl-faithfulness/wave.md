Title: Sql Schema Ddl Faithfulness

## Objective

Raise SQL schema-DDL extraction faithfulness: stop minting phantom `reads` edges for custom column types, `DEFAULT`/`GENERATED` expression tokens, pseudo-type params, and `FOR UPDATE SKIP LOCKED`; model `CREATE TYPE`/`CREATE DOMAIN` as definition nodes; and recognize `CREATE OPERATOR`. On a type-dense schema the phantom count is proportional to column count, so this is a measurable precision gain over the current statement unit.
