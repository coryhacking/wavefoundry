## Objective

**Cut the 1.11.0 release (operator direction 2026-07-06 — one combined release wave).** Two dashboard lifecycle defects plus the release vehicle. `1rswx` — the MCP server spawns the dashboard as an un-reparented child it never reaps, so dead dashboards accumulate as zombies and `wave_dashboard_stop`/`restart` fail on them (bare-`os.kill` liveness can't tell a zombie from a live process); `1rtju` — the live dashboard silently stops reflecting repo changes because its single watcher thread can wedge with no timeout/watchdog, its directory-mtime watch misses nested-file edits, the client has no recovery when SSE is "connected" but silent, and MCP-spawn discards the logs that would reveal it. `1rtvf` — upgrade forces a full graph rebuild when `GRAPH_BUILDER_VERSION` advances (so consumers get this cycle's SQL graph accuracy + Oracle/T-SQL dialect work) + documents the post-upgrade MCP-reload requirement, and carries VERSION → 1.11.0 + CHANGELOG + package. When this wave closes: stop/restart reliable on dead instances, the running dashboard stays fresh (or visibly flags), upgrades materialize the new graph extraction, and 1.11.0 is cut + packaged.
