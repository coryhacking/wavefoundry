Title: Framework Drift Convergence — Eliminate Dual-Valid States And Unenforced Claims

## Objective

Close the structural drift class field-surfaced by the Solaris upgrade (1.5.0+p3d6) and the operator's "no dual-valid states, no unenforced claims" framing. The framework currently tolerates ambiguity by design — renames accept both spellings indefinitely, `MUST` claims in seeds are not enforced by `docs-lint`, and every canonical name lives in four places that can independently disagree (seed prose, consumer runtime, lint alias tables, hand-authored docs). The drift gate honestly returns `ok` because both truths are permitted. This wave eliminates the ambiguity at the source: every rename converges to one name within a bounded window; every `MUST` is enforced or downgraded; every canonical name has exactly one source of truth that every other surface derives from. Ships jointly under the **1.5.0** tag with closed waves `1p35d` and `1p3b9`.
