Title: Install Experience Hardening

## Objective

Reduce install friction surfaced by a real 1.9.8 native-Windows install, so the fixes ship **in 1.10.0**
(release held for this wave). The journal/persona/manifest validators reject artifacts for requirements
the seeds never state and with errors that name the rule but not the expected value — costing the install
agent 2–3 round-trips per artifact — and one disallowed pattern even blocks a journal from *describing*
its own rule. Close both sides (feed-forward seed contracts + feedback error text), fix the pattern
false-positive, and stop the recurring empty-`applicable_factors` audit warning.
