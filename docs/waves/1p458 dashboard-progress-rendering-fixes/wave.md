Title: Dashboard Progress Rendering Fixes

## Objective

Two dashboard rendering corrections surfaced from operator use. (1) Long change IDs in the Activity "recent changes" list force the whole panel wider because the id is rendered `white-space: nowrap`; they should wrap at dash boundaries (never at the kind↔slug space). (2) The progress graph (ACS/TASKS bars) reports intentionally-deferred (`[~]`) ACs and tasks as a separate "· N deferred" tally outside the completed total; deferred items should count toward the completed total once the wave is **closed**, and read as **outstanding** (in the denominator, not done) while the wave is **open** — with the separate tally removed in both states. Closes when both ship with tests.
