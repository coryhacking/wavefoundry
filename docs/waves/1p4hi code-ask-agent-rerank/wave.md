Title: Code Ask Agent Rerank

## Objective

Make `code_ask`'s default retrieval **agent-fused** and **structure-aware**. Skip the ~30s local cross-encoder (`bge-reranker-base`) **and** the context-blind RRF cross-source merge, returning **labeled, deduped, per-index candidates** (full chunk text, with a per-index coverage floor) for the host agent — the tool's only consumer, which already synthesizes — to fuse, rank, and synthesize itself. **Five changes** (sequence `1p4hj` → `1p4mf` → `1p4lr` → `1p4ls` → `1p4hu`): `1p4hj` (the agent-fuse pipeline — default `rerank="agent"`; cross-encoder+RRF retained as opt-in `"local"`; gated on AC-10 — **landed, PASS**); **`1p4mf`** (chunk module-level constant declarations so a constant's value is retrievable at all — the **foundational** fix; `CHUNKER_VERSION` 25→26, gated on a local recall eval before fleet re-index); `1p4lr` (a candidate-side definition/constant rank boost — makes the now-retrievable but short constant chunk *rank*; **revived by `1p4mf`** and shipped coupled to it — the one class AC-10 found agent losing to the cross-encoder); `1p4ls` (graph **constant nodes + `reads` edges** — the structural complement: constants are absent from the graph today, blocking structural "what value is X" + constant blast-radius + `1p4hu`'s reach to constants; the one `GRAPH_BUILDER_VERSION` bump in the wave); and `1p4hu` (a structural **graph signal** as a candidate source so call-graph-relevant code surfaces where text similarity misses it). Two surfaces, one fix: chunks make constant values **retrievable**, the graph makes them **structural**. Cuts per-call latency + the shared-model parallel bottleneck and makes retrieval structure-aware.

## Wave Summary

Five-change wave reworking `code_ask` retrieval + making it structure-aware. **`1p4hj` (agent-fuse, default):** `code_ask` already does no synthesis (the agent synthesizes), so the cross-encoder is a redundant ~30s/contention step — and RRF is *also* a context-blind rank heuristic the agent can out-fuse. Agent-mode (the new default) skips **both**: it returns labeled per-index candidates (each index's top-N, budget-allocated with a per-index floor so no modality is starved, deduped for diversity while preserving the cross-source-agreement signal, as **full chunk `text`** — reranker-parity, never vectors) and the agent fuses+ranks+synthesizes. Cross-encoder+RRF are retained as the opt-in `"local"` mode (eval/determinism/offline); the default flip is **gated** on a quality+recall check (`1p4hj` AC-10) confirming agent-mode matches-or-beats it. **`1p4hu` (graph signal):** adds a structural candidate source — 1-hop call/import/community neighbors of the query's resolved symbol — so call-graph-relevant code surfaces where embedding similarity misses it (generalizing today's keyword two-hop), bounded + with a no-graph fallback, gated on its own quality check. `1p4hj` + `1p4hu` are query/response-layer only. **`1p4lr`** (added 2026-06-10) adds a candidate-side definition/constant rank boost — the query-layer lever for the "what value/model is X" class `1p4hj` AC-10 found agent losing to the cross-encoder (`RERANKER_MODEL` rank 11 vs 3). **`1p4ls`** (added 2026-06-10) adds graph **constant nodes + `references` edges** — the structural complement (constants are absent from the graph; `code_definition("RERANKER_MODEL")` → `graph_definitive_not_found`), and the **one `GRAPH_BUILDER_VERSION` bump (25 → 26)** in the wave; it gives `1p4hu` constant edges to traverse. **`1p4mf`** (added 2026-06-10) is the **foundational** prerequisite the others assumed — it makes module-level constants *chunks* at all (today they are in zero chunks: `chunk_python` emits only func/class/docstring), so `code_ask` can retrieve a constant's value semantically; per-constant `UPPER_SNAKE` shape with a `_merge_small_chunks` guard, `CHUNKER_VERSION` 25→26 **gated on a local recall eval** (the heaviest, irreversible lever — prove it before fleet re-index), shipped in **one combined re-index window** with `1p4ls`'s graph bump. The prepare-council (2026-06-10) **verified** `1p4lr` could not move its motivating query under the no-chunk status quo (constants weren't chunked); `1p4mf` supplies the substrate and **revives** `1p4lr` (chunk = retrievable, boost = ranks). **Sequence: `1p4hj` (landed, AC-10 PASS) → `1p4mf` → `1p4lr` → `1p4ls` → `1p4hu`** (`1p4hu` consumes both the new chunk lane and `1p4ls`'s edges + plugs into the per-index allocation).

## Journal Watchpoints

- **Shared-file serialization (blocking):** both `1p4hj` and `1p4hu` rework `code_ask` candidate selection in `server_impl.py` (the per-index retrieval + `_rerank:833`/`_rrf_merge:851` region). **Land `1p4hj` first** — it establishes the per-index allocation + `source` labels + dedup that `1p4hu` plugs its graph source into. Sequence the edits; coordinate one mode-branch.
- **Agent fuses — skip cross-encoder AND RRF (design core):** agent-mode (the default) returns labeled per-index candidates (full chunk `text`, per-index floor, deduped), NOT a cross-encoder rescore or an RRF cross-source order. Dedup must **preserve the multi-source-agreement signal** (RRF's cross-source reward) — keep the source set, don't collapse to one.
- **Chunks not vectors:** full chunk `text` per candidate (reranker parity, `:838` — not the 300-char excerpt `:14395`); NEVER embedding vectors (uninterpretable to the agent + token-wasteful; server-internal only).
- **Default-flip gated (blocking):** agent-mode is the DEFAULT — the flip ships only if it matches-or-beats `"local"` (cross-encoder+RRF) on quality+recall (`1p4hj` AC-10). `"local"` retained for eval/determinism/offline; do NOT remove the cross-encoder or RRF.
- **Token budget (blocking):** every default call returns the per-index set → bound by a token/size budget (mirror the `1p4es` `code_impact` cap); the levers are the per-index counts + the cross-index allocation (≈ budget ÷ chunk-size), not a gut number.
- **Version-bump scope — TWO bumps (updated 2026-06-10):** `1p4hj`, `1p4lr`, `1p4hu` are query/response-layer (`server_impl.py`); `1p4hu` *consumes* the existing graph (`graph_query.py`). **`1p4ls`** changes graph node/edge shape (constant nodes + `reads` edges) → **`GRAPH_BUILDER_VERSION` 25→26**. **`1p4mf`** changes chunk shape (constant chunks) → **`CHUNKER_VERSION` 25→26**, which forces a **FULL code-corpus re-index** (auto-escalation, indexer.py:1990-2019) — strictly heavier than the graph rebuild and the heaviest lever in the pipeline. **Land both bumps in ONE combined release** so consumers pay a single re-index window; gate the `CHUNKER_VERSION` bump on a local AC-10 recall eval before it reaches consumers (see [[graph-builder-version-bump]]). Edit gate: `framework_edit_allowed`.
- **Sequence + shared-file (updated 2026-06-10):** `1p4hj` (landed) → `1p4mf` → `1p4lr` → `1p4ls` → `1p4hu`. **`1p4mf`** edits `chunker.py` (constant-chunk emission + `CHUNKER_VERSION` bump) — independent file, lands **FIRST** of the constant work as the substrate `1p4lr` boosts. `1p4hj`, `1p4lr`, `1p4hu` all rework the `search_combined` agent-branch candidate scoring/selection in `server_impl.py` — land in order, coordinate the one region. `1p4ls` edits `graph_indexer.py` (independent file) but its version bump forces a full graph rebuild; sequence it before `1p4hu` so `1p4hu` can traverse the new constant `reads` edges. `1p4lr`'s AC-5 value gate runs **after** `1p4mf` lands (it now has real constant chunks to move).
- **`1p4hu` bounded + fallback:** graph expansion capped (1-hop default) within the budget; graceful no-graph degrade to `1p4hj` semantic-only.
