Title: Post Release Field Hardening

## Objective

Close two field-feedback gaps surfaced against the shipped **1.6.0+p5lj** release. (1) On NVIDIA hosts with CUDA 13 (Arch/CachyOS/Manjaro, RTX 40xx/50xx), `onnxruntime-gpu`'s CUDA-12 ABI pin makes the GPU provider fail to load and index builds **silently** run on CPU (5–10× slower); the framework must either auto-shim the `.so.12→.so.13` mismatch (root-free, venv-local) or **surface the failure + remediation to the install/upgrade agent** — never fall back silently. (2) A real secret was mislabeled `false-positive` because the `confirmed-secret` status (which already gives "real, upgrade-allowed, ship-blocked-until-fixed") wasn't clearly documented; clarify it so agents/operators classify correctly. When this wave closes, GPU fallback is loud or fixed, and the secrets status workflow is unambiguous.
