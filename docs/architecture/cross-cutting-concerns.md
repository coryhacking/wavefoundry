# Cross-Cutting Concerns

Owner: Engineering
Status: active
Last verified: 2026-05-08

## Configuration

- **Primary config:** `docs/workflow-config.json` ‚Äî lifecycle epoch, wave settings, review policies, and dashboard host/port preferences. Read by `lifecycle_id.py`, `dashboard_server.py`, and the Wave Framework self-hosted surface.
- **Guard overrides:** `.wavefoundry/guard-overrides.json` (gitignored) ‚Äî temporary flags for seed/framework edits. Consumed by Claude Code pre-edit hook.
- **Dashboard runtime metadata:** `.wavefoundry/dashboard-server.json` (host-local; untracked) ‚Äî last known dashboard endpoint and startup metadata. Written only by `dashboard_server.py`.
- **Dashboard auto-index write boundary:** when `dashboard.auto_index: true`, `dashboard_server.py` may also write to `.wavefoundry/index/` via a spawned indexer subprocess. This is the only non-metadata write the daemon performs; it is opt-in, subprocess-isolated (`start_new_session=True`, `close_fds=True`), and uses the same atomic temp-then-rename strategy as direct indexer runs.
- **Platform config:** `.claude/settings.json`, `.cursor/hooks.json`, `.github/hooks/hooks.json` ‚Äî generated by `render_platform_surfaces.py`; committed to version control.
- **Future MCP config:** format TBD; will include `allowed_roots`, `default_root`, `index` settings (see AGENTS.md Configuration Sketch).

All layers may access: `docs/workflow-config.json`.
Only `render_platform_surfaces.py` writes to platform config files.

## Logging / Output

- Framework scripts log to stdout/stderr (human-readable). No structured logging system yet.
- `docs_lint.py` exits non-zero on failure with actionable error messages.
- MCP server: structured JSON responses (MCP protocol).
- Dashboard server: browser assets plus JSON snapshot responses over loopback HTTP; prints the final bound URL on startup.

## Error Handling

- Framework scripts: exit non-zero on failure; print error to stderr. No retry logic.
- MCP server: return structured error responses per MCP protocol; never silently ignore failures.
- Dashboard server: fail fast when no configured port is available; browser renders an explicit unavailable state rather than fabricating data.

## Shared Utilities

- `.wavefoundry/framework/scripts/wave_lint_lib/` ‚Äî shared modules for docs_lint: `link_validators.py`, `metadata_validators.py`, `context.py`, `helpers.py`.
- These are internal to `.wavefoundry/framework/scripts/`; not exposed as a public library.

## Observability

No external monitoring, alerting, or distributed tracing. The local dashboard is an operational read surface, not a telemetry pipeline.
MCP server and dashboard server may both want richer stderr request logging later; current behavior stays intentionally minimal.
