# Cross-Cutting Concerns

Owner: Engineering
Status: active
Last verified: 2026-04-28

## Configuration

- **Primary config:** `docs/workflow-config.json` — lifecycle epoch, wave settings, review policies. Read by `lifecycle_id.py` and the Wave Framework self-hosted surface.
- **Guard overrides:** `.wavefoundry/guard-overrides.json` (gitignored) — temporary flags for seed/framework edits. Consumed by Claude Code pre-edit hook.
- **Platform config:** `.claude/settings.json`, `.cursor/hooks.json`, `.github/hooks/hooks.json` — generated by `render_platform_surfaces.py`; committed to version control.
- **Future MCP config:** format TBD; will include `allowed_roots`, `default_root`, `index` settings (see AGENTS.md Configuration Sketch).

All layers may access: `docs/workflow-config.json`.
Only `render_platform_surfaces.py` writes to platform config files.

## Logging / Output

- Framework scripts log to stdout (human-readable). No structured logging system yet.
- `docs_lint.py` exits non-zero on failure with actionable error messages.
- Future MCP server: structured JSON responses (MCP protocol).

## Error Handling

- Framework scripts: exit non-zero on failure; print error to stderr. No retry logic.
- Future MCP server: return structured error responses per MCP protocol; never silently ignore failures.

## Shared Utilities

- `framework/scripts/wave_lint_lib/` — shared modules for docs_lint: `link_validators.py`, `metadata_validators.py`, `context.py`, `helpers.py`.
- These are internal to `framework/scripts/`; not exposed as a public library.

## Observability

No monitoring, alerting, or distributed tracing (local tools; no operational dashboard needed).
Future MCP server: may want request/response logging to stderr for debugging; TBD.
